Methods Used (Step by Step)

Each method below is shown with where it was applied and what artifact it produced.

science

Experimental Design

UAV Teleop Fog

Designed controlled conditions and tasks to compare interface variants and user performance.

Output: Study structure, conditions, scenarios, procedures

speed

Performance Metrics

Teleop Fog UAV (task + SA)

Measured response time, errors, and task outcomes to evaluate interface impact.

Output: Logged events, RT tables, error counts, performance graphs

analytics

Statistical Analysis

UAV Teleop Fog

Tested hypotheses and compared conditions (t-tests / ANOVA depending on design).

Output: p-values, test statistics, result tables used in the slides

draw

Prototyping & UI Iteration

UAV Teleop Fog

Built and refined interfaces, then validated them through controlled evaluation loops.

Output: Final UI screens, tested variants, interaction flows

This shows the methods as an end-to-end pipeline: design → measure → test → iterate. The same rigorous workflow was applied across three different safety-critical contexts.
